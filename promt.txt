Bạn là chuyên gia Android/Java.  
Tôi có một đoạn code quản lý menu game (Back menu + Sub menu Advanced).  
Nhiệm vụ:
1. Đọc code, cho tôi biết thứ tự hiện tại của các tính năng trong Back menu và Sub menu (Advanced) trong file app/src/main/java/com/limelight/GameMenu.java
2. Sửa code để sắp xếp lại menu theo thứ tự sau:

Back menu (menu chính):
1. Disconnect (Ngắt kết nối game stream)
2. Hiển thị gamepad (Toggle Virtual Model / Virtual Controller)
3. Hiển thị bàn phím mềm (Toggle Keyboard)
4. Hiển thị bàn phím đầy đủ (Toggle Virtual Keyboard Model)
5. Hiển thị phím đặc biệt (Send Keys)
6. Chọn chế độ chuột (Select Mouse Mode)
7. Bật Pan/Zoom (Enable/Disable Zoom Mode)
8. Xoay màn hình (Rotate Screen)
9. Tùy chọn nâng cao (Advanced)
10. Thoát phiên (Quit Session)
11. Hủy (Cancel)

Sub menu Advanced:
1. Hiển thị lớp phủ hiệu suất (Toggle HUD)
2. Thay đổi độ nhạy cảm ứng (Switch Touch Sensitivity Model)
3. Gửi nội dung clipboard (Upload Clipboard)
4. Lấy nội dung clipboard (Fetch Clipboard)
5. Lệnh máy chủ (Server Cmd)
6. Mở trình Task Manager (Task Manager = Ctrl+Shift+Esc)
7. Gửi phím đặc biệt (Send Keys)
8. Hủy (Cancel)

Yêu cầu khi sửa code:
- Giữ nguyên logic gốc (if/else, check context, allowChangeMouseMode...).
- Chỉ thay đổi **thứ tự addItem()** trong back menu và submenu.
- Không làm mất các điều kiện hiển thị (ví dụ: chỉ hiển thị Rotate nếu context = game).
- Xuất code sau khi chỉnh sửa, không giải thích thêm.





Bạn là một trợ lý lập trình Android cấp cao.

Nhiệm vụ khởi đầu:
1. Đọc toàn bộ dự án Android mình cung cấp (bao gồm Gradle, Manifest, source code Java/Kotlin, XML layout, resource).
2. Phân tích và ghi chú lại:
   - Tên package chính và entry point (ví dụ MainActivity, Application).
   - Kiến trúc dự án (MVC, MVVM, Clean Architecture... nếu có).
   - Các Activity/Fragment/Composable quan trọng.
   - Các file layout chính liên kết với logic.
   - Thư viện và dependency trong build.gradle.
   - Cách dữ liệu/setting đang được quản lý.
3. Cho biết dự án này là về cái gì, ứng dụng có tác dụng gì (dựa trên code, package, layout, resource).
4. Giữ nguyên coding style và convention hiện tại.
5. Không đề xuất thay đổi hoặc tối ưu gì ở bước này.

Kết quả mong đợi:
- Tóm tắt dự án: app này dùng để làm gì, phục vụ mục đích nào.
- Tóm tắt kiến trúc dự án, luồng hoạt động chính, và cách các thành phần liên kết với nhau.
- Liệt kê các file quan trọng (tên + chức năng).
- Sau bước này, bạn sẽ ở trạng thái "đã hiểu dự án" và sẵn sàng thực hiện mọi task chỉnh sửa hoặc thêm tính năng mà mình yêu cầu.
